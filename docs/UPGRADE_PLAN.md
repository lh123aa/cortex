## 一、严重Bug修复 (P0 - 必须修复)

### 1.1 HNSW搜索变量错误
| 项目 | 内容 |
|------|------|
| **文件** | `internal/vector/hnsw.go` |
| **行号** | 347 |
| **问题** | 搜索循环使用 `entryPoint` 而非 `current`，导致图遍历错误 |
| **修复** | `h.neighbors[level][entryPoint]` → `h.neighbors[level][current]` |
| **影响** | 搜索结果可能完全错误 |
| **工时** | 10分钟 |

### 1.2 SQL语法错误
| 项目 | 内容 |
|------|------|
| **文件** | `internal/storage/search.go` |
| **行号** | 101 |
| **问题** | `WHERE c.id IN (''' + joinStrings(chunkIDs) + ''')` 三引号非法 |
| **修复** | 使用 `strings.Join()` 拼接ID，移除三引号 |
| **影响** | 向量搜索必定失败 |
| **工时** | 15分钟 |

### 1.3 认证数据持久化
| 项目 | 内容 |
|------|------|
| **文件** | `internal/auth/service.go` + `internal/storage/crud.go` |
| **问题** | 用户/token/APIKey全存内存map，服务重启丢失 |
| **修复** | 实现 User/Token/APIKey 的 SQLite CRUD |
| **影响** | 用户需重新注册，API Key需重新生成 |
| **工时** | 4-6小时 |

### 1.4 统计方法实现
| 项目 | 内容 |
|------|------|
| **文件** | `internal/storage/crud.go` |
| **行号** | 306-339 |
| **问题** | `GetChunksCount()`、`GetVectorsCount()` 等返回0 |
| **修复** | 实现真实COUNT查询 |
| **影响** | 统计API返回错误数据 |
| **工时** | 30分钟 |

## 二、核心功能完善 (P1)

### 2.1 L1内存缓存集成
| 项目 | 内容 |
|------|------|
| **文件** | `internal/search/engine.go` |
| **问题** | `SearchCache` (L1) 已定义但从未调用 |
| **修复** | 在 `Search()` 中集成 L1 → L2 两级缓存 |
| **效果** | 缓存命中率预计提升 30-50% |
| **工时** | 1-2小时 |

### 2.2 Config热重载修复
| 项目 | 内容 |
|------|------|
| **文件** | `internal/config/config.go` |
| **行号** | 203-218 |
| **问题** | `watch()` 方法每秒轮询但不触发 `OnConfigChange` |
| **修复** | 实现真正的 `fsnotify` 文件监听 |
| **效果** | 修改config.yaml无需重启服务 |
| **工时** | 1小时 |

### 2.3 记忆删除缓存失效
| 项目 | 内容 |
|------|------|
| **文件** | `internal/api/memory.go` |
| **行号** | 321 |
| **问题** | 删除记忆后未清空搜索缓存 |
| **修复** | 删除后调用 `InvalidateSearchCache()` |
| **效果** | 避免搜索返回已删除记忆 |
| **工时** | 15分钟 |

### 2.4 Embedding维度配置化
| 项目 | 内容 |
|------|------|
| **文件** | `cmd/cortex/main.go` |
| **行号** | 154 |
| **问题** | 硬编码 `768` |
| **修复** | 从配置读取或动态获取 |
| **效果** | 支持不同维度embedding模型 |
| **工时** | 30分钟 |

### 2.5 用户上下文nil保护
| 项目 | 内容 |
|------|------|
| **文件** | `internal/api/auth_middleware.go` |
| **问题** | `GetUserContext()` 可能返回nil导致panic |
| **修复** | 所有handler检查nil或返回默认游客身份 |
| **效果** | 避免认证失败时服务崩溃 |
| **工时** | 1小时 |
