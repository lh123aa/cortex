## 二、性能测试

### 2.1 代码性能分析

#### 向量搜索性能

```go
// storage/search.go - VectorSearch
func (s *SQLiteStorage) VectorSearch(...) {
    // HNSW 搜索: O(log n)
    // 暴力搜索: O(n)
}
```

**评估**:
- HNSW参数: M=32, MaxLayers=16, EfConstruction=200
- 搜索复杂度: O(log n) 预期
- 10万向量估计延迟: 10-50ms

#### Embedding 生成性能

```go
// embedding/ollama.go - EmbedBatch
func (e *OllamaEmbedding) EmbedBatch(texts []string) {
    // 4并发处理
    // 批量大小: 无限制
}
```

**评估**:
- 并发数: 4 (ants pool)
- Ollama延迟: 50-100ms/请求
- 10个文本: 200-400ms

#### 索引性能

```go
// index/indexer.go
pool, _ := ants.NewPool(cfg.Workers)  // 默认8 workers
```

**评估**:
- Worker数: 8 (可配置)
- 吞吐量预估: 100-200 files/min

### 2.2 性能瓶颈分析

| 组件 | 瓶颈 | 影响 | 建议 |
|------|------|------|------|
| Ollama Embedding | 网络延迟 | 50-100ms/请求 | 本地部署 Ollama |
| SQLite写入 | WAL同步 | 批量写入优化 | 已使用批量插入 |
| HNSW构建 | 内存 | 大规模向量 | PQ量化压缩 |
| 缓存 | 内存 | 缓存淘汰 | 已有过期机制 |
